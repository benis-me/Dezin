# Cursor

> Category: Developer Tools

## 1. Visual Theme & Atmosphere

A dark-native code editor: deep charcoal surfaces, a calm blue accent, monospace
where it counts. Focused and low-glare for long sessions.

## 2. Color Palette & Roles

- Surfaces: --bg #1b1b1f, --surface #232328, --surface-2 #2c2c33
- Text: --fg #e6e6ea, --fg-2 #b3b3bd, --muted #8a8a96
- Accent: --accent #4da6ff on --accent-fg #0b0b0e
- Border: --border #34343c, --border-strong #45454f

## 3. Typography Rules

UI sans + a strong monospace for code/paths. Body 13–14px; mono for identifiers.
Tight, dense lines; ALL-CAPS labels at 0.06em.

## 4. Component Stylings

Buttons: subtle --surface-2 fill or --accent for primary, radius 6px. Inputs: 1px
border, focus → --accent. Code: --surface tint, mono. No shadows in the editor body.

## 5. Layout Principles

8px grid; dense rows; panels separated by 1px borders. Radius 4 / 6 / 8px.

## 6. Depth & Elevation

Flat panels. Command palettes/menus float with one soft dark shadow.

## 7. Do's and Don'ts

Do: stay dark, dense, quiet; one blue accent. Don't: light-first layouts, neon, pure-black voids, multiple accents.

## 8. Responsive Behavior

Breakpoints 768 / 1200px; collapse side panels under 768; targets ≥24px.

## 9. Agent Prompt Guide

bg #1b1b1f / fg #e6e6ea / accent #4da6ff / border #34343c. Example: "A dark settings panel with mono shortcut hints and a blue primary."
