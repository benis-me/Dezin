# GitHub

> Category: Developer Tools

## 1. Visual Theme & Atmosphere

Pragmatic, information-dense developer UI: clean white (or dim dark) surfaces, a
trustworthy blue accent, and lots of bordered list rows. Function over flourish.

## 2. Color Palette & Roles

- Light: --bg #ffffff, --surface #f6f8fa, --fg #1f2328, --muted #59636e, --border #d1d9e0, --accent #0969da
- Dark: --bg #0d1117, --surface #161b22, --fg #e6edf3, --muted #8b949e, --border #30363d, --accent #2f81f7
- Status: --success #1a7f37, --danger #cf222e

## 3. Typography Rules

System sans; mono for code/SHAs. Body 14px / 1.5; headings 16–24px. Two weights.

## 4. Component Stylings

Buttons: --surface fill + 1px border, or --accent primary; radius 6px. Cards/boxes:
1px --border, no shadow. Labels/badges: pill, small. Code: --surface tint, mono.

## 5. Layout Principles

8px grid; bordered list rows; container max 1280px. Radius 6 / 8px.

## 6. Depth & Elevation

Flat; borders define everything. Overlays (menus, dialogs) get one soft shadow.

## 7. Do's and Don'ts

Do: borders + rows, one blue accent, tabular-nums for counts. Don't: gradients, drop shadows on cards, a second accent.

## 8. Responsive Behavior

Breakpoints 768 / 1012 / 1280px; stack columns under 768; targets ≥24px.

## 9. Agent Prompt Guide

light bg #fff / fg #1f2328 / accent #0969da / border #d1d9e0. Example: "A repo file list: bordered rows, blue links, mono SHAs, a green status pill."
