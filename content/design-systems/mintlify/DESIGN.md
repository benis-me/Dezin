# Mintlify

> Category: Docs & SaaS

## 1. Visual Theme & Atmosphere

Friendly, airy documentation: bright white reading surface, a fresh teal-green
accent, rounded-but-restrained components, and a calm three-column docs layout.

## 2. Color Palette & Roles

- Surfaces: --bg #ffffff, --surface #f7faf9, --surface-2 #eef4f2
- Text: --fg #0f1d1a, --fg-2 #44524e, --muted #6b7a75
- Accent: --accent #14b8a6 on --accent-fg #04211d
- Border: --border #e3ebe8, --border-strong #cbd8d3

## 3. Typography Rules

Clean sans for prose; mono for code. Body 16px / 1.6; headings with clear scale.
Measure ~72ch in the content column.

## 4. Component Stylings

Sidebar nav with a teal active state; code blocks with a soft tint + copy; callouts
as a left-rule + tint (not a colored card border). Buttons radius 8px.

## 5. Layout Principles

Three zones (nav · content · on-this-page). 4px grid; radius 6 / 8 / 12px.

## 6. Depth & Elevation

Flat; light shadow only on hover cards and menus.

## 7. Do's and Don'ts

Do: airy reading, one teal accent, clear nav highlight. Don't: heavy borders, neon, multiple accents, cramped lines.

## 8. Responsive Behavior

Breakpoints 768 / 1100px; collapse sidebar to a menu under 768; targets ≥24px.

## 9. Agent Prompt Guide

bg #fff / fg #0f1d1a / accent #14b8a6 / border #e3ebe8. Example: "A docs page: teal active nav item, tinted code blocks, left-rule callouts."
