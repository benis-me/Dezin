# Changelog

A single `index.html` changelog that's easy to skim top-to-bottom.

## Structure

- Reverse-chronological entries. Each: a date (tabular-nums), an optional version tag, a one-line summary, then grouped changes.
- Tag change types subtly (Added / Fixed / Changed) with a small label — distinguished by weight or a quiet pill, not a rainbow of colors.
- A left date rail or hanging dates create a clear timeline spine.
- Keep each entry tight; link out for detail rather than dumping everything.

## Craft

Real, specific entries — what changed and why it matters — never "Various improvements".
One accent, used only for version tags or links. Hairline dividers between entries; no cards/shadows.
